Operation

Операции одноразовые связаны с выделением|освобождением ресурсов

Операция подготовки процесса к выполнению заключается в формировании блока описания процесса, выделение виртуальной памяти, ... . После формирования контекста процесса иногда приходится формировать контекст регистровый (значение регистров начальное).

Операция завершения работы заключается в ликвидации контектста процесса. Необходимо освободить ресурсы, ассоциированные с процессом, запрошенные динамически(файлы, устройства), виртуальную память.

Thread - процесс легкий, дешовый , без виртуальной памяти. Нет накладных расходов. Создать thread дешевле чем процесс с контекстом полным.

Описатели процессов закончившихся продолжают оставаться в системе время некоторое.

Операции многоразовые не приводят к изменению числа процессов в OS, не связаны с выделением|особождением ресурсов.

OS выбирает процесс для выполнения из списка процессов, находящихся в состоянии "готовность", на основании ее политики. Для запуска процесса OS обеспечивает наличее данных в оперативной памяти необходимых для выполнения процесса.

Состояние выбранного процесса изменяется на "исполнение". Значения регистров процессора восстанавливаются на основании PCB. Управление передается команде, на которую указывает счетчик команд. Работа процесса в стадии "испольнение" может быть приастановлена из-за прерываний. При этом процессор сохраняет счетчик команд, регистры в стеке процесса, передает управление аппаратное по адресу обработки прерывания.
