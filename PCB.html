PCB

Процесс
соввокупность набора команд исполняющихся, ресурсов, ассоциированных с ним (память, пространство адресное, стеки, файлы, устройства ввода|вывода) , текущего контекста (значения регистров процессора, счетчика команд, адреса команды, переменных, состояние стека) находящаяся под управлением OS.
Process Control Blok (PCB)
Структура данных, которая представляет в OS процесс.

Процесс представляется в OS структурой данных или блоком управления процессом (PCB).

PСB содержит:

  • cостояния управления процесса: "блокирован", "готов", "исполняется";
  • значения регистров процесса в момент остановки, включая счетчик процесса;
  • данные для планирования использования процессора и управления памятью (приоритет, размер и расположение адресного пространства, ...);
  • учетные данные: (process.id, пользователь, время исползования процессора);
  • информация об устройствах ввода-вывода, связанных с процессом: таблица открытых файлов, таблица устрйств,

Один процесс может выполнять несколько программ.

Несколько процессов могут выполнять одну программу.

Процесс управляется OS.

Процесс может выполнять часть кода OS.

Все что выполнияется в OS, организуется как набор процессов.

В системе компьютерной однопроцессорной в каждый момент выполняется только один процесс. В системе вычислительной мультипрограммной обеспечивается одновременная обработка процессов нескольких.

Изменением состояний процессов занимается OS. Набор операций:

  • создание (одноразовая),
  • приостановка|запуск,
  • блокировка|разблокировка.

Процесс новый рождается в состоянии "готов". OS его выбирает, переводит в состояние "исполнение". Процесс выполняется на процессоре. У него 3 исхода:

  • окончание,
  • прерывание по таймеру,
  • готовность.

Готовность и ожидание состояния разные. Процессору из состояния готовность можно выбрать процесс, а из состояния ожидание нет.

Для появления в системе вычислительной процесс должен пройти состояние "рождение". В этом состоянии процесс полчает в свое распоряжение от OS адресное пространство, в которое загружается код программный, выделяется стек, ресурсы системные, устанавливается значение счетчика команд или счетчика процесса программного. После этого он становится процессом полноправным и переводится в состояние "готовность".

Процесс в состоянии "исполняется" может завершиться или приостановиться OS и переведен в состояние "не исполняется".

Причины приостановки процесса:

  • событие (завершение ввода|вывода)
  • решение OS(истечение времянного интевала).

Процесс помещается в состояние "не исполняется" пока не будет выбран OS. Состояние "не исполняется содержит два подсостояния

  • "готовность"
  • "ожидание".

Чтобы сделать процесс несуществующим, нужно забрать у него все ресурсы и вернуть их OS. Этому соответствует состояние "закончил исполнение".

Процесс блокируется, когда он не может продолжить свою работу, не дождавшись какого- либо события.

Процесс обращается к OS с помощью вызова системного, который не дает возврата пока не будет выполнена операция внешняя.

OS обрабатывает вызов системный: инициализирует операцию ввода-вывода, добавляет процесс в очередь процессов, дожидающихся освобождения устройства или возникновения события.

Ядро сохраняет необходимую часть контекста процесса в его PCB и переводит процесс в состояние «ожидание».

После возникновения в системе события (завершения операции ввода-вывода, ожидания поступления сообщения) OS определяет какое событие произошло.

Система проверяет, находился ли процесс в состоянии «ожидание» для данного события, если находился, то переводит его в состояние «готовность» (то есть для него условие продолжения работы выполнено). Перед этим, в зависимости от события, выполняются действия дополнительные, связанные с наступлением события.

Планрование рескрсов.

OS при работе в режиме мултипрограммном имеет ограничение на ресурсы: память, процессоры. Все остальное виртуализируется. Необходимо распределять ресурсы между потребителями - планировать. Планирование должно опираться на критерии, алгоритмя, параметры потребителя.

Уровнии планирования:

  • I: задания - процедура принятия решения, о том, какое задание для загрузки выберется из пакета заданий, то есть для какого задания породить процесс;
  • II: процессора - процедура выбора процесса, который получить процессор в свое распоряжение. Возникает в системах мультипроцессорных, где в состоянии "готовность" могут находиться несколько процессов;

Выбор задания порождает образование процесса на уровене планирования процесса.

Планирование заданий - планирование процессов долгосрочное, так как отвечает за порождение процессов в системе, определяя степень мультипрограммирования (число процессов, одновременно находящихся в системе).

Степень мультипрограммирования неограниченная - пока хватит ресурсов.

Степень мультипрограммирования постоянная - не более чем. Процессы образуются после завершения процессов образованных.

Решение о выборе процесса для запуска на процессоре (выборе задания, для которого создается процесс) оказывает влияние на функционироване вычислительной системы на протяжении интервала длительного времени.
